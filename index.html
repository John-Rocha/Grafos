Bloco 3 · Definições

O que é um Grafo?

Um grafo é uma estrutura formada por um conjunto de vértices e um conjunto de arestas, usada para representar relações entre elementos.

Notação formal

G = (V, E)

V — vértices  ·  E — arestas

Exemplo

V = {A, B, C, D}   |V| = 4

E = {AB, BC, CD, DA, AC}   |E| = 5

Definições

Vértice e Aresta

Vértice (Nó)

Representa um ponto, elemento ou entidade do problema.

Ex.: cidade, pessoa, página web, gene

Aresta (Ligação)

Representa a conexão ou relação entre dois vértices.

Ex.: estrada, amizade, hiperlink, reação

Definições

Grau de um Vértice

O grau de um vértice é o número de arestas que incidem sobre ele.

A deg(A) = 3
B deg(B) = 2
C deg(C) = 2
D deg(D) = 1

Soma dos graus = 2 × |E|

Cada aresta contribui com 2 graus

Por que Representar Grafos no Computador?

Duas representações principais: Matriz de Adjacência  ·  Lista de Adjacência
Representações

Matriz de Adjacência

Uma matriz n × n onde M[i][j] = 1 se existe aresta entre i e j, e 0 caso contrário.

1234
10110
21011
31100
40100

M[i][j] = 1 → existe aresta entre i e j

Representações

Matriz de Adjacência — Análise

✓ Vantagens

  • Simples de implementar
  • Verificação de aresta em O(1)
  • Boa escolha para grafos densos

✗ Desvantagens

  • Consome O(V²) de memória
  • Ineficiente para grafos esparsos
  • Custo alto para grafos muito grandes
Memória O(V²)
Verificar aresta O(1)
Listar vizinhos O(V)

Lista de Adjacência — Análise

✓ Vantagens

  • Memória O(V + E) — muito econômica

Implementação — Matriz de Adjacência

1

adicionarAresta(u, v)

Define M[u][v] = M[v][u] = 1

2

imprimirMatriz()

Percorre e exibe a matriz n × n

3

calcularGrau(v)

Soma todos os elementos da linha v

#define MAX 10
int adj[MAX][MAX];
int n; /* nº de vértices */

void adicionarAresta(int u, int v) {
  adj[u][v] = 1;
  adj[v][u] = 1;
}

int calcularGrau(int v) {
  int grau = 0;
  for (int i = 0; i < n; i++)
    grau += adj[v][i];
  return grau;
}
Prática em C

Implementação — Lista de Adjacência

1

adicionarAresta(u, v)

Insere v na lista de u (e u na de v)

2

imprimirLista()

Percorre cada lista encadeada

3

calcularGrau(v)

Conta os nós na lista de v

struct No {
  int dest;
  struct No *prox;
};

struct No *adj[MAX];

void adicionarAresta(int u, int v) {
  /* cria nó e insere na lista */
  inserir(&adj[u], v);
  inserir(&adj[v], u);
}

int calcularGrau(int v) {
  struct No *p = adj[v];
  int g = 0;
  while (p) { g++; p = p->prox; }
  return g;
}

Escolhendo a Estrutura Certa

Matriz de Adjacência

Use quando o grafo é denso ou quando precisa verificar conexões rapidamente

O(1) por consulta
vs

Lista de Adjacência

Use quando o grafo é esparso ou quando a memória é limitada

O(V+E) de memória
Ambas resolvem o mesmo problema — a escolha depende do tipo de grafo e do algoritmo a ser aplicado.
